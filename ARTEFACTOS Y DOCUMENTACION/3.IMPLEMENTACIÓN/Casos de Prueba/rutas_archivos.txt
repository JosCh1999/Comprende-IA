# Ubicación de Archivos Clave del Proyecto

Este documento detalla la ubicación de los entregables requeridos por la rúbrica para las pruebas de la API.

## 1. Mock API con JSON Server

La implementación del servidor de API simulada se encuentra en la carpeta `mock/`.

- **Archivo de Datos (Base de Datos Falsa):**
  - Ubicación: `mock/db.json`
  - Descripción: Contiene los datos de ejemplo para las entidades `usuarios` y `textos`.

- **Archivo de Configuración y Ejecución:**
  - Ubicación: `mock/package.json`
  - Descripción: Contiene el script para iniciar el servidor mock. Para ejecutarlo, navega a la carpeta `mock` y ejecuta `npm install` seguido de `npm start`.

## 2. Pruebas Unitarias y de Integración con Supertest

Las pruebas automatizadas para los endpoints de la API, desarrolladas con Jest y Supertest, están centralizadas en la carpeta de pruebas del backend.

- **Carpeta Principal de Pruebas:**
  - Ubicación: `backend/tests/`
  - Descripción: Contiene todos los archivos de especificación de pruebas (`.test.js`).

- **Pruebas de Autenticación:**
  - Ubicación: `backend/tests/auth.test.js`
  - Descripción: Pruebas para los endpoints `/auth/register` y `/auth/login`.

- **Pruebas de Textos:**
  - Ubicación: `backend/tests/textos.test.js`
  - Descripción: Pruebas para los endpoints `POST /textos`, `GET /textos` y `GET /textos/:id`, incluyendo la validación de rutas protegidas.
